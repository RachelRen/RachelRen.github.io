概念

它最大的特点是，服务器可以主动向客户端推送信息，客户端也可以主动向服务器发送信息，是真正的双向平等对话，属于服务器推送技术的一种。

它的特点

  1. 建立在TCP协议上，服务端的实现比较容易
  2. 与HTTP协议有着良好的兼容性。默认端口也是80和443， 并且握手阶段采用HTTP协议，因此握手时不容易屏蔽，能通过各种HTTP代理服务器。
  3. 数据格式比较轻量，性能开销小，通信效率高。
  4. 可以发送文本，也可以发送二进制数据
  5. 没有同源限制，客户端可以与任意服务器通信
  6. 协议标识符是ws(如果加密，则为wss)，服务器网址就是URL

基本属性

属性 类型 说明
readyState Object 返回实例对象的当前状态，一共有4种
onopne EventListener 指定连接成功后的回调函数
onclose EventListener 关闭后的回调函数
onmessage EventListener 收到服务器后的数据回调
onerror EventListener 报错时的回调函数
send function 用于向服务器发送数据
close function 关闭此次连接
bufferedAmount 返回数字 表示还有多少字节的二进制数据没有发送出去。可以用来判断发送是否结束

向服务器发送的数据类型可以是以下几种:

USVString, ArrayBuffer, Blob, ArrayBufferView

readyState
|值|说明|
|—|—|
|CONNECTING|值为0，表示正在连接|
|OPEN|值为1， 表示成功连接，可以通信了|
|CLOSING|值为2， 表示正在关闭|
|CLOSED|值为3，表示已经关闭连接，或者打开连接失败|
