AMD (Asynchronous Module Definition)

异步模块定义

API:

  1. id 第一个参数id为字符串类型，表示模块标识，可选。如果没有提供参数，模块的名称应该默认为模块加载器请求的指定脚本的名称。如果提供了该参数，模块名必须是”顶级”的和绝对的
  1. dependencies array类型。包含一组当前模块依赖的，已被模块定义的模块标识。此参数也是可选的。如果忽略此参数，它应该默认为["require", "exports", "module"].
  2. factory 函数（工厂方法），模块初始化要执行的函数或对象。如果是函数，它应该只被执行一次。如果是对象，那么该对象是这个模块的输出值。

应用场景

  • 无依赖：

    ​

  • 有依赖的模块

  • 定义对象模块

  • 全参的

  • 输出模块的

    输出模块的格式和nodejs是一样的，使用require获取模块，使用exports导出API

    除了defined外，AMD还保留了一个关键字require。require作为规范保留的全局标识。

CMD(common module definition)

通用模块定义

它推崇依赖就近，想什么时候require 就什么时候加载，实现了懒加载(延迟执行)，没有全局require,每个API都简单纯粹。一个模块就是一个文件。

factory 为函数时，表示模块的构造方法。这个构造方法可以得到模块向外提供的接口。在执行factory方法时，默认会传入三个参数: require, exports, module

CMD 和 ADM 的区别

  1. 对于依赖的模块，AMD是提前执行，CMD是延迟执行。不过RequireJS 从2.0开始，也改成了可以延迟执行(根据写法不同，处理方式不同)。CMD推崇as lazy as possible.
  2. CMD推崇依赖就近，AMD推崇依赖前置。

CommonJs

CommonJS是服务器端模块的规范，NodeJS采用了这个规范。

commonJS，一个单独的文件就是一个模块。加载模块使用require方法，该方法读取一个文件并执行，最后返回文件内部的exports对象。

CommonJS加载模块是同步的，所以只有加载完成后，才能执行后面的操作。因为nodeJS主要用于服务器编程，需要加载的模块文件一般都已经存在本地硬盘，所以加载起来比较快，不需要考虑异步加载的方式。但是如果是浏览器环境，需要从服务端加载模块，就必须采用异步模式。所以才有了AMD CMD解决方案。
