##基础知识

react是把真实的DOM树转换成javascript对象树，也就是Virtual DOM,当每次数据更新之后，都会重新计算Virtual DOM, 并和之前生成的Virtual DOM做对比。虽然现在都在说虚拟DOM提升性能，但是对比DOM节点也是需要计算资源的，虽然比传统的渲染方式要好一点，但是并不很明显。

函数式编程式React的精髓。

React中创建的虚拟元素可以分两类：DOM元素和组件元素，分别对应着原生的DOM元素与自定义元素。

无状态组件

只传入props和context两个参数，没有state, 也没有生命周期方法。无状态组件不是在调用周期会创建实例，它创建时始终保持了一个实例，避免了不必要的检查和内存分配，做到了内部优化。

数据流

如果顶层组件初始化props改变，那么React会向下遍历整棵组件树，重新尝试渲染所有相关组件。state只关心每个组件自己内部的状态，这些状态也只在组件内部改变。

React.Children是官方提供的一系列操作children的方法。它提供了map， forEach， count等函数，可以为我们处理子组件提供了便利。

React生命周期

从广义上定义，可以分为挂载，渲染，卸载这几个阶段。
React生命周期分为两类：

  1. 当组件在挂载和卸载时
  2. 当组件接收新的数据时，即组件更新时

挂载：componentWillMount和componentDidMount， 前面一个是在render方法之前执行，后面那个是在render之后执行。

卸载：componentWillUnmount,组件卸载前的状态，用来在这个方法中回收和清除定时器。

更新：

  • componentWillReceiveProps(nextProps)
  • shouldComponentUpdate(nextProps, nextState)
  • componentWillUpdate(nextProps, nextState)
  • componentDidUpdate(prevProps, prevState)

如果是state改变，那么只会执行后面3中方法。

无状态组件是没有生命周期的，所以每次都会重新渲染。

React DOM

这整个插件提供的API非常少，只有findDOMNode, unmountComponentAtNode, render.

只有在componentDidMount和componentDidUpdate中才将真正的DOM插入到HTML中。

findDOMNode和refs都无法用到无状态组件，无状体组件挂载时只是方法调用，没有新建实例。

React事件绑定

react中实现了虚拟DOM的合成事件层，它主要做了事件委派和自动绑定。

事件委派是指react将所有事件都绑定到结构的最外层，使用一个统一的事件监听，这个事件监听器上维持了一个映射来保存所有组件内部的事件监听和处理函数。当事件发生时，先处理这个统一的事件监听器，然后进行映射。这样做的好处是简化了事件处理和回收，效率有所提高。

因为用了事件委派，是一个整体的过程，react对上下文进行了缓存处理，所以当在一个组件中调用自己对象时，需要手动绑定this。

React事件中，是没有捕获阶段的，因为兼容性。
