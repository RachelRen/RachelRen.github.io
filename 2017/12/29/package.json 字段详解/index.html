package.json 字段详解

2017-12-29

每个项目的根目录下，一般都有一个package.json文件，定义了这个项目所需要的各种模块，以及项目的配置信息（比如名称，版本，许可证等元数据）。npm install命令根据这个配置文件，自动下载所需的模块，也就是配置项目所需的运行和开发环境。

name

  1. 都是小写，
  2. 只能是一个词，不能有空格
  3. _和-都是允许的

version

  • name

    包的名字，命名规则：都是小写，数字，-,_, 禁止出现空格；包名是唯一的（npm库里的包名）

  • version

    版本号

  • description

    该包的简介

  • main

    入口文件
    如果不存在这个字段，会自动按下面的顺序查找: index.js, index.node, index.json

  • scripts

    指定了运行脚本命令的npm命令行缩写，比如start指定了运行npm run start时，所要执行的命令。

  • keywords
  • author
  • license
  • bugs
  • homepage

  • dependencies

    通过npm install <package_name> --save来安装组件，而且这些组件是开发这个组件所需要的

  • devDependencies

    通过npm install <package_name> --save-dev来安装的，这些组件只在开发的时候所需要依赖的。
